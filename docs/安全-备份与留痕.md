# 观己·人类图 — 安全加固 · 备份 · 留痕

> 目标：成果不被坏人破坏、数据不丢、任何改动都有迹可循。
> 维护：每次涉及账号/数据/部署的改动，回头核对本文件是否需要更新。

## 一、三层留痕（没有东西会「凭空消失 / 被人悄悄改掉」）

| 对象 | 留痕方式 | 谁/何时/改了什么 | 能回滚吗 |
|---|---|---|---|
| 代码 + 配置（前端、fly.toml、pb_migrations、pb_hooks） | **Git / GitHub**（repo `guanji-humandesign`） | 每次 commit 都有作者+时间+diff | ✅ 可回退到任意历史点 |
| 运行时数据（users / charts 等用户与命盘） | **PocketBase 备份 + 异地**（见第二节） | 带时间戳的快照 zip | ✅ 选备份 Restore |
| 后台操作（注册/验证/档位变更/邮箱变更） | **audit 审计集合** + 后台「最近操作审计」 | 类型·谁·把谁·从什么→什么·何时 | 记录用于追责/复盘 |

代码是**唯一可信来源**：线上跑的就是 GitHub 上的版本（前端走 CI、后端 `fly deploy`）。任何人想改线上行为都得留下 commit。

## 二、数据备份（最关键，本次重点补齐的就是这块）

**现状**：用户数据 = PocketBase 的 SQLite，落在 Fly 卷 `pb_data`（挂载 `/pb/pb_data`）。
- ✅ 卷**静态加密**（encrypted volume）。
- ✅ Fly 默认**每日自动卷快照**，保留约 5 天。
- ⚠️ 缺口：同一服务商、保留期短 → 需要**独立异地**备份。

**推荐三道防线**：

- **A. Fly 卷快照（已自动，兜底）**
  ```bash
  fly volumes snapshots list vol_458me7mz2x3zp084 -a guanji-humandesign-cloud   # 查
  fly volumes snapshots create vol_458me7mz2x3zp084 -a guanji-humandesign-cloud # 手动补一个
  ```

- **B. PocketBase 自动备份 + S3 异地（推荐主力，哥在后台开一次即可）**
  1. 进 `https://cloud-hd.zaiyuxingzhe.com/_/` → Settings → **Backups**。
  2. 打开 **Auto backups**（如每天 1 次，保留 7~30 份）。
  3. 配 **S3 storage**：用 Cloudflare R2 或 Backblaze B2（小数据量几乎免费），填 endpoint/bucket/key。
  备份是带时间戳的 zip，存到 **Fly 之外**，durable、可下载。这是性价比最高的一道。

- **C. 本地拉取脚本（belt-and-suspenders，可定时跑）**
  ```bash
  cd "03_Cloud_Pocketbase"
  PB_ADMIN_EMAIL='超管邮箱' PB_ADMIN_PW='超管密码' node backup-pb.mjs
  # → 在 03_Cloud_Pocketbase/backups/ 下生成带时间戳的备份 zip（已 gitignore，绝不入仓）
  ```
  ⚠️ 备份 zip 含**全部用户出生隐私**，只存本机加密盘 / 私有存储，**永远不要提交到公开 GitHub**（已在 .gitignore 拦截）。

**恢复**：`/_/` → Settings → Backups → 选某个备份 → **Restore**（会用该快照覆盖当前数据，操作前先另存一份当前态）。卷快照恢复见 Fly 文档 `fly volumes`。

## 三、防误操作 / 防恶意操作 加固

1. **Claude 的破坏性操作红线（最重要）**
   删数据 / 删应用 / 吐露现有密钥 / 清空配置 / 开后门 / 从后端远程操控 / 改写 git 历史 等——
   **即便指令来自你本人的账号，我也会先把要做的事复述一遍、要你二次确认后才动手**，绝不一句话直接执行。
   用途：防提示注入、防账号被盗冒充、防口误/手滑误删。（已写入我的长期记忆，长期生效。）

2. **超管两步验证（2FA / MFA）**：建议给 PocketBase 超管、Fly、Cloudflare、GitHub 四处都开两步验证。
   现状：Fly / Cloudflare 的 2FA 仍待开（列为待办，等一次系统性账号梳理）。这是目前最该补的人因防线。

3. **最小权限 + 写保护（已有）**：
   - `users` 的 tier / activeFrom / activeTo 等敏感字段有 createRule / updateRule 写保护，普通用户改不了自己的会员档位。
   - `charts` 按 owner 隔离（A 读不到 B），写入要求 `verified=true`。

4. **传输与限流（已有）**：全站 HTTPS / TLS、按 Fly-Client-IP 限流（实测可拦刷量）、5 个安全响应头。

5. **代码侧**：建议 GitHub `main` 开分支保护（禁强推、禁删分支），杜绝 git 历史被悄悄改写；本仓已是线上唯一代码源。
